# 票务内测客服与运营异常处理手册

## 目的

这份手册给客服和运营提供一套统一的异常处理底线口径，适用于第一次内测开卖窗口。

当订单没有走在 happy path 上时，就按这里来。目标是两件事：`用户侧口径准确`，`运营侧 case 始终有人接住`。

## 共通规则

1. 在出票结果真正确认前，不能告诉用户“已经出票”。
2. 在最终退款回调落地前，不能告诉用户“退款已经完成”。
3. 每一笔异常单都必须有一个明确负责人。
4. 每一笔异常单至少记录这些证据：
   `orderId`、`orderNumber`、后台可见状态、小程序可见状态、当前负责人。
5. 如果涉及支付，还要补充微信 `out_trade_no` 和支付交易号。
6. 如果涉及出票，还要补充上游事件标识和票码。
7. 如果涉及退款，还要补充退款单号。

## 场景 1：支付成功但未出票

### 触发条件

用户已经完成支付，但订单仍然在等待出票确认。

### 运营检查动作

1. 确认订单出现在后台 `/orders`。
2. 确认订单出现在后台 `/fulfillment`。
3. 检查 `POST /api/payments/wechat/callback` 是否已经成功。
4. 检查该订单是否已经提交给上游。
5. 如果支付回调成功后超过 `15` 分钟仍无出票进展，立刻转人工跟进。

### 用户回复锚点

“您的支付已经完成，目前出票结果还在确认中，所以这笔订单暂时还不能视为已出票。我们会继续跟进出票结果，并在确认后第一时间通知您。”

### 结案条件

只有当订单明确进入已出票，或者已经给到用户确认的替代处理路径时，才能结案。

## 场景 2：上游出票失败或延迟

### 触发条件

上游明确拒绝出票、长时间超时，或者没有返回可用的出票回调。

### 运营检查动作

1. 记录订单 ID。
2. 记录后台和小程序当前可见状态。
3. 记录已经返回的上游事件标识或票码（如果有）。
4. 判断下一步是继续人工催出票，还是切入退款路径。
5. 记录负责人和下次更新时间。

### 用户回复锚点

“这笔订单的出票没有正常完成，目前已经转人工核查，我们正在跟进上游返回结果，并会在承诺的时间内给您下一次更新。”

### 结案条件

只有当出票完成、退款处理接管，或者已经给用户明确确认的恢复路径时，才能结案。

## 场景 3：退款处理中

### 触发条件

退款申请已经受理，但最终退款回调还没有完成。

### 运营检查动作

1. 确认请求出现在后台 `/refunds`。
2. 记录退款单号。
3. 确认退款请求已经发送到上游。
4. 根据退款单号或订单 ID 检查上游回调日志。

### 用户回复锚点

“您的退款申请已经受理，目前正在等待上游确认。我们会持续跟进退款结果，直到这笔退款最终完成。”

### 结案条件

只有在最终退款完成状态可见后，才能结案。

## 场景 4：退款超时或退款失败

### 触发条件

退款没有顺利完成，或者检测到上游退款失败。

### 运营检查动作

1. 确认请求出现在后台 `/refunds`。
2. 记录退款单号和当前状态。
3. 检查上游退款提交日志。
4. 检查上游退款回调日志。
5. 让这笔 case 保持在退款异常队列中可见。

### 用户回复锚点

“这笔退款目前还在处理中，因为上游确认没有正常完成。我们已经对该 case 做了升级跟进，会持续追踪，直到给您一个最终结果。”

### 结案条件

只有退款完成，或者进入明确批准的替代处理方案时，才能结案。

## 场景 5：重复回调或状态不一致

### 触发条件

支付回调或上游回调重复出现，或者可见状态互相矛盾。

### 运营检查动作

1. 确认订单当前状态。
2. 记录重复回调的时间和 payload 标识。
3. 确认订单没有从终态回退。
4. 如果后台状态和小程序订单详情已经不一致，立即升级处理。

### 用户回复锚点

“这笔订单目前存在系统侧状态核对中的情况，我们已经在跟进，并会以最终确认状态为准向您同步后续结果。”

### 结案条件

只有后台视图和用户视图状态重新一致后，才能结案。

## 场景 6：用户身份信息错误

### 触发条件

观演人信息填写有误，或导致信息无法成功录入。

### 运营检查动作

1. 判断订单是否仍处于出票前。
2. 判断问题是否发生在演出前最后 `3` 天内。
3. 判断原因是否为 `USER_IDENTITY_ERROR`。
4. 如果三者都成立，则按规则扣除 `20%` 服务费。
5. 将扣费依据写入 case 备注。

### 用户回复锚点

“这笔订单属于观演人信息错误处理场景。由于问题发生在已公布的临近演出窗口内，所以退款结果会按规则扣除 `20%` 服务费。”

### 结案条件

只有当退款或恢复结果已经可见，并且用户已经明确知晓规则依据时，才能结案。
